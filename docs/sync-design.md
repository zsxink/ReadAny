# 多后端同步方案设计

## 一、现有架构分析

当前同步系统基于 WebDAV，包含：
- **数据库同步**：整体覆盖同步（VACUUM INTO 快照）
- **文件同步**：书籍文件 + 封面图片
- **冲突检测**：通过 manifest 比较决定同步方向

## 二、新增同步后端

### 1. S3 兼容存储同步

```
┌─────────────────────────────────────────────────────────────┐
│                     S3 Sync Adapter                          │
├─────────────────────────────────────────────────────────────┤
│ 支持服务商：                                                  │
│   • AWS S3                                                   │
│   • Cloudflare R2 (无出站费用)                                │
│   • 阿里云 OSS (S3 兼容)                                      │
│   • 腾讯云 COS (S3 兼容)                                      │
│   • MinIO (自建)                                             │
│                                                              │
│ 配置项：                                                      │
│   • endpoint: S3 端点 URL                                    │
│   • region: 区域                                             │
│   • bucket: 存储桶名称                                        │
│   • accessKeyId: 访问密钥                                    │
│   • secretAccessKey: 密钥                                    │
│   • pathStyle: 是否使用路径样式 URL                          │
└─────────────────────────────────────────────────────────────┘
```

**优势**：
- 成本低（尤其 R2 无出站费用）
- 速度快（CDN 加速）
- 稳定可靠

**劣势**：
- 需要配置云服务
- 需要网络连接

### 2. 局域网临时同步

```
┌─────────────────────────────────────────────────────────────┐
│                    LAN Sync Adapter                          │
├─────────────────────────────────────────────────────────────┤
│ 工作模式：                                                    │
│   • 发现模式：扫描局域网内的 ReadAny 实例                     │
│   • 服务端模式：在本机启动 HTTP 服务，等待连接                │
│   • 客户端模式：连接到其他设备的 HTTP 服务                    │
│                                                              │
│ 连接方式：                                                    │
│   • 扫码连接：二维码包含 IP、端口、配对码                     │
│   • 手动连接：输入 IP 地址 + 配对码                           │
│   • 自动发现：mDNS/Bonjour 扫描局域网                        │
│                                                              │
│ 安全机制：                                                    │
│   • 临时配对码（6位数字）                                     │
│   • TLS 加密传输                                             │
│   • 一次性会话密钥                                            │
└─────────────────────────────────────────────────────────────┘
```

**优势**：
- 无需云服务
- 传输速度快（局域网带宽）
- 适合临时同步场景

**劣势**：
- 需要设备在同一局域网
- 需要同时在线

## 四、统一同步接口设计

```typescript
// packages/core/src/sync/sync-backend.ts

/** 同步后端类型 */
export type SyncBackendType = 'webdav' | 's3' | 'lan';

/** 统一同步后端接口 */
export interface ISyncBackend {
  /** 后端类型标识 */
  readonly type: SyncBackendType;
  
  /** 测试连接是否可用 */
  testConnection(): Promise<boolean>;
  
  /** 确保远程目录结构存在 */
  ensureDirectories(): Promise<void>;
  
  /** 上传数据 */
  put(path: string, data: Uint8Array): Promise<void>;
  
  /** 下载数据 */
  get(path: string): Promise<Uint8Array>;
  
  /** 获取 JSON 数据 */
  getJSON<T>(path: string): Promise<T | null>;
  
  /** 上传 JSON 数据 */
  putJSON<T>(path: string, data: T): Promise<void>;
  
  /** 列出目录内容 */
  listDir(path: string): Promise<RemoteFile[]>;
  
  /** 删除文件 */
  delete(path: string): Promise<void>;
  
  /** 获取显示名称（用于 UI） */
  getDisplayName(): Promise<string>;
}

/** 远程文件信息 */
export interface RemoteFile {
  name: string;
  path: string;
  size: number;
  lastModified: number;
  isDirectory: boolean;
}
```

## 十、安全考虑

### LAN 同步安全

1. **配对码机制**：6位数字配对码，有效期5分钟
2. **传输加密**：使用 TLS 1.3 加密所有传输数据
3. **会话密钥**：每次连接生成一次性会话密钥
4. **设备绑定**：首次配对后记录设备指纹，防止中间人攻击

### S3 同步安全

1. **凭证存储**：使用系统密钥链存储 accessKeyId 和 secretAccessKey
2. **传输加密**：强制使用 HTTPS
3. **最小权限**：建议用户配置只读/只写权限的 IAM 策略
4. **桶策略**：建议配置桶策略限制访问 IP

## 十一、错误处理

### 网络错误

- 连接超时：提示用户检查网络
- DNS 解析失败：提示用户检查地址
- SSL 证书错误：提示用户检查证书

### 同步冲突

- 数据库冲突：显示冲突对话框，让用户选择保留哪个版本
- 文件冲突：以较新版本为准，保留本地备份

### 存储空间不足

- 同步前检查本地存储空间
- 显示需要同步的数据大小
- 提供选择性同步选项（仅数据库/包含文件）
